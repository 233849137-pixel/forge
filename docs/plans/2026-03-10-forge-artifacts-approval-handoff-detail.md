# Forge 工件页补齐 approval handoff detail

## 目标

- 让工件页 `正式工件责任` 与首页、项目页、治理页保持同一层级的 `approvalHandoff` 可见性。
- 避免负责人在工件工作台里只能看到 `确认后接棒 / 接棒动作`，却看不到来源链细节。

## 本批完成

- `ForgeArtifactsPage` 的 `正式工件责任` 现在会直接显示 `接棒细节`。
- 工件页四段责任摘要现在完整覆盖：
  - `确认后接棒`
  - `接棒细节`
  - `接棒动作`
- 页面断言已补到 `forge-os-pages.test.tsx`，验证工件页会显示：
  - 接棒来源命令
  - 接棒来源运行
  - 接棒 runtime label

## 结果

- `首页 / 项目页 / 工件页 / 治理页` 现在都直接显示同一条 `approvalHandoff` 责任链。
- 负责人不需要离开工件工作台，就能判断“确认后谁接棒、来自哪条外部执行链、下一步做什么”。
