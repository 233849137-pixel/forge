# Forge 节点页信息架构基线

日期：2026-03-08
状态：已落地到当前桌面端

## 目标

Forge 不再使用“一个首页 + 大量并列卡片”的管理后台结构，而是采用“首页驾驶舱 + 节点专页”的产品形态。

本文件用于约束后续页面演进：

1. 首页只负责判断项目当前卡在哪一步
2. 节点页只承载该节点必须处理的动作和上下文
3. 不允许把创建项目、Prompt、PRD、执行、门禁、沉淀同时堆到一个页面

## 页面分工

### 首页

定位：驾驶舱，不是工作台全集。

允许出现：
- 节点轨道
- 当前节点工作台
- 风险与阻塞
- 最近执行

禁止出现：
- 新建项目表单
- Prompt 模板库
- PRD 生成
- 推荐复用资产
- 团队编制

### 项目接入

定位：建立项目，不做方案和执行。

允许出现：
- 当前项目集
- 快速建立项目
- 模板绑定

禁止出现：
- Prompt 模板库
- PRD 草案
- 推荐复用资产
- 执行记录

### 方案与任务包

定位：收敛方案边界，产出可执行输入。

允许出现：
- Prompt 模板库
- PRD 生成
- 最新 PRD 草案

禁止出现：
- 项目创建
- 推荐复用资产
- 风险阻塞
- 团队编制

### 开发执行

定位：推进编码和补丁，不展示接入和沉淀模块。

允许出现：
- 当前节点工作台
- 执行焦点
- 执行队列

禁止出现：
- 风险与阻塞
- Prompt 模板库
- PRD 生成
- 推荐复用资产

### 测试验证

定位：只围绕门禁和阻塞转绿。

允许出现：
- 当前节点工作台
- 风险与阻塞
- 验证记录

禁止出现：
- 项目创建
- Prompt 模板库
- 推荐复用资产
- 归档内容

### 交付发布

定位：整理交付材料和发布状态，不再混入执行流水。

允许出现：
- 当前节点工作台
- 交付概要
- 最新 PRD 草案

禁止出现：
- 最近执行
- 验证记录
- 推荐复用资产

### 归档复用

定位：沉淀可复用知识，不处理当前执行动作。

允许出现：
- 沉淀建议
- 推荐复用资产
- 最新 PRD 草案

禁止出现：
- 当前节点工作台
- 风险与阻塞
- 执行队列

### 团队

定位：Agent Team 编制页，不和节点页混排。

允许出现：
- 当前团队
- 工件流转
- 角色分工

禁止出现：
- 项目创建
- PRD 生成
- 测试门禁
- 推荐复用资产

## 模块复用规则

允许跨页复用的模块：
- 当前节点工作台
- 最新 PRD 草案
- 风险与阻塞
- 执行记录面板

复用要求：
- 标题必须跟随页面语义变化
- 同一信息不能在一个页面里重复出现两次
- 复用模块不能携带不属于该节点的操作入口

## 下一步实现约束

后续新增功能时，按以下顺序落位：

1. 先判断它属于哪个节点
2. 再决定放在首页、节点页还是团队页
3. 若跨节点使用，必须抽成独立模块并定义标题变化规则
4. 若会导致首页新增一个大面板，默认视为错误设计
