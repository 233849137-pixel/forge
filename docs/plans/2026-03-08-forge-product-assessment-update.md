# Forge 产品评估更新

日期：2026-03-08

## 结论

旧评估里有三类内容：

1. 已经完成的
- 本地 SQLite 持久化
- PRD 草案生成
- MCP / 本地 HTTP AI 调用层
- macOS 桌面壳与 Electron 启动链

2. 仍然成立的
- 真实 Codex / Claude 执行仍未接入
- Prompt 模板还没有编辑、版本回滚、审批
- Skill 还缺真正执行引擎
- 多人协作、历史版本、云同步仍未实现

3. 需要按当前产品重新表述的
- Forge 已不再是“节点页看板”，现在主轴是 `Agent Team OS`
- 一级导航已经切成 `首页 / 项目 / 团队 / 工件 / 执行 / 资产 / 治理`
- 项目与资产之间已补了显式 `projectAssetLinks`

## 本轮优化

### 1. 信息架构
- 首页只保留负责人驾驶舱
- 项目页承接阶段推进与项目建立
- 资产页独立承载 Prompt / Skill / 模板 / 门禁关联

### 1.1 产品表达
- 首页标题已改成 `本地 AI 交付流水线`
- 项目页标题已改成 `项目起盘与推进`
- 产品叙事从“Agent Team OS 骨架”收口成“把 AI 交付从聊天式推进变成可执行、可追踪、可复用的本地流水线”

### 2. 关系模型
- 为项目新增显式资产关联 `projectAssetLinks`
- 关联对象覆盖模板、Prompt、Skill / 资产
- 创建项目时自动生成默认关联

### 2.1 起盘注入
- 创建项目时会自动注入首批工件：`PRD / 架构说明 / 原型与交互规范 / TaskPack`
- 创建项目时会自动注入首批任务：`需求确认 / PRD / 原型架构 / 本地执行与门禁`
- 本地工作区会自动生成 `README / project-dna / intake` 三类起盘文件

### 3. 资产页
- 新增 `当前项目推荐`
- 新增 `Skill 使用指引`
- 新增 `关联模板与门禁`
- 保留 `Prompt 模板库` 与 `可复用资产`

## 当前产品判断

Forge 当前更准确的定义是：

`一个 local-first 的 Agent Team OS 骨架，已经完成一级结构和基础关系建模，但还没有完成真实执行引擎。`

## 下一优先级

1. 接入真实 Runner，把 Codex / Claude / Playwright 从记录层变成执行层
2. 给 Prompt 模板补编辑、版本、变量表单
3. 给工件补历史与变更流
4. 给团队补 handoff 规则和待接棒队列
